Color Usage Guidelines

Recommended Usage
  • Primary Blue: Main actions, highlights, key metrics, and charts
  • Success Green: Positive indicators, confirmations, profitable metrics
  • Danger Red: Errors, warnings, negative metrics, deletions
  • Warning Yellow: Cautions, mid-range indicators
  • Info Teal: Informational elements, secondary actions
Accessibility Considerations
  • Maintain contrast ratio of at least 4.5:1 for normal text
  • Maintain contrast ratio of at least 3:1 for large text
  • Don't rely solely on color to convey information
  • Provide additional indicators (icons, text) alongside color

Typography Guidelines

Best Practices
  • Maintain proper heading hierarchy (h1 → h6)
  • Use consistent line height (1.5 for body, 1.2 for headings)
  • Limit line length to 60-80 characters for readability
  • Use system fonts for optimal performance
  • Bold for emphasis, not entire paragraphs
Accessibility Considerations
  • Minimum text size of 14px for all content
  • Don't use justified text alignment
  • Avoid all-caps text for long content
  • Provide adequate contrast between text and background
  • Allow text to be resized up to 200% without loss of content

Chart Guidelines

Chart Configuration
  • Colors: Use consistent colors from the application color palette
  • Grid Lines: Light grid lines (rgba(255, 255, 255, 0.1))
  • Text: Labels and ticks in light gray (rgba(255, 255, 255, 0.7))
  • Tooltips: Dark background with light text and data values
  • Legend: Position at top or right side of charts
  • Animation: Subtle animations for initial rendering (duration ≤ 1000ms)
Accessibility Considerations
  • Provide alternative text descriptions for all charts
  • Use patterns in addition to colors for distinguishing data series
  • Ensure sufficient contrast between chart elements
  • Include data tables as alternatives to visual charts where appropriate
  • Use ARIA attributes to improve screen reader accessibility
  • Support keyboard navigation for interactive chart elements

Data Tables

DataTables implementation with consistent styling, including:

  • Pagination controls
  • Search/filter functionality
  • Sorting indicators
  • Row selection

// Initialize DataTable with consistent styling
const table = new DataTable('#resultsTable', {
    order: [[5, 'desc']], // Sort by score by default
    pageLength: 10,
    lengthMenu: [[10, 25, 50, 100, -1], [10, 25, 50, 100, "All"]],
    columnDefs: [
        {
            targets: 0, // Select checkbox column
            orderable: false,
            className: 'select-checkbox',
            render: function() {
                return '<div class="form-check"><input class="form-check-input select-row" type="checkbox"></div>';
            }
        },
        {
            targets: -1, // Actions column
            orderable: false,
            render: function() {
                return '<button class="btn btn-sm btn-outline-primary"><i class="fas fa-plus"></i></button>';
            }
        }
    ],
    responsive: true,
    language: {
        search: "Filter:",
        emptyTable: "No results available."
    }
});
